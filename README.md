# via-ppt-rescue-maker

一个用于 Codex 的 PPT 页面生成 skill。

它适合把主题、粗略大纲、完整文档、散乱素材、旧 PPT 或数据表，整理成一套可以汇报的 PPT 页面 PNG。工作流会先确认页序和内容逻辑，再生成两张风格样张，用户确认后才继续批量生成剩余页面。

## 它解决什么问题

很多 PPT 失败不是因为不够漂亮，而是因为还没想清楚：

- 这次到底要讲什么
- 听众会问什么
- 每一页承担什么任务
- 哪些内容来自材料，哪些只是推测或待核验
- 页面应该像真实汇报材料，而不是海报、模板样机或素材拼贴

这个 skill 的目标是先把内容讲述结构定住，再把每页转成高质量 16:9 PNG 页面。

## 核心原则

- 先定内容，再生成图片
- 默认不生成 PPTX，只交付页面 PNG 和演讲提纲
- 默认走两次确认：先确认页序，再确认两张风格样张
- 学术、数据、复盘等场景必须区分来源状态、假设和待核验信息
- 每页都要能被真实拿去汇报，而不是只有视觉效果

## 适合场景

- 组会汇报
- 开题、答辩、课程汇报
- 内部复盘
- 数据复盘
- 内容策略汇报
- 企业内训或工作坊
- 从旧 PPT 或散乱素材重做一套更清楚的页面

## 使用方式

把这个仓库放到 Codex 的 skills 目录后，在 Codex 里这样调用：

```text
用 $via-ppt-rescue-maker 把我的主题或材料整理成能讲的 PPT。
```

你也可以直接给它主题、文档、大纲、数据表或旧 PPT，并说明：

- 汇报场景
- 听众是谁
- 大概页数
- 汇报时长
- 是否需要学术引用或数据依据

## 工作流

1. 判断材料状态：主题、大纲、完整文档、散乱素材、旧 PPT 或数据表
2. 确认场景契约：使用场合、听众、页数、时长、证据密度和内容边界
3. 输出页序与内容锁定草案，等待用户确认
4. 生成第 1 页和一张代表性内容页作为风格样张
5. 用户确认风格后，批量生成剩余页面 PNG
6. 最终交付页面图片和演讲提纲

## 输出物

默认最终只展示：

```text
页面图片/
演讲提纲.md
```

引用与假设说明通常合并在 `演讲提纲.md` 末尾。只有在用户明确要求，或正式学术答辩、报告引用很多时，才单独生成 `引用与假设说明.md`。

## 注意事项

- 这个 skill 不负责凭空编造论文、DOI、样本量、年份或精确数据
- 没有用户材料或可靠来源时，会把内容标为模拟假设或待核验
- 样张确认前不会提前生成整套页面
- 每页成品应是独立、完整、真实尺寸的 16:9 PPT 页面 PNG
