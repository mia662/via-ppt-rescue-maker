## 材料分流与场景契约

页序与内容锁定草案是第一个用户可见确认物，不是第一步逻辑。输出页序草案之前，先完成材料分流和场景契约。

材料状态只能先归入以下六类之一：

| 材料类型 | 先补什么 |
| --- | --- |
| 只有主题 | 场景、具体问题、主线、证据边界 |
| 有大纲 | 把目录改成听众问题链和页面任务 |
| 有完整文档 | 从阅读结构重组为口头汇报结构 |
| 有散乱素材 | 先分事实、判断、目标和缺口 |
| 有旧 PPT | 先诊断页面角色、重复页、缺失页和证据缺口 |
| 有数据表 | 先定义决策问题、指标含义、对比关系和行动建议 |

然后确定场景契约：

- 使用场合：组会、开题/答辩、课程汇报、内部复盘、数据复盘、内容策略、企业内训等
- 听众身份：导师、评审、同学、执行者、决策者、客户、家长等
- 页数和预计时长
- 证据密度：低、中、高，或按场景描述
- 内容边界：这次只讲什么，不讲什么
- 来源边界：用户材料、待核验引用、需要计算的数据、模拟假设
- 最终交付：页面 PNG + 演讲提纲/讲稿

## 状态机

按以下状态执行，不能跳步：

1. `待确认页序`：先完成材料分流与场景契约，再输出页序与内容锁定草案，不生成图片。
2. `待确认首图`：只生成第 1 页和代表性内容页样张，停止等待用户确认。
3. `批量生成中`：用户确认样张后，生成剩余页面图片。
4. `最终交付`：交付 `页面图片/` 和 `演讲提纲.md`。

如果当前状态是 `待确认首图`，完成条件不是“图片已生成”，而是“用户已看过并明确确认继续”。

## 第一次确认

第一次确认是“页序与内容锁定草案”。它是第一个给用户看的确认物；在它之前已经完成材料分流和场景契约。不要生成图片，不要只输出页标题。

必须包含：

- 当前不生成图片
- 材料状态：六类之一
- 先补策略：根据材料状态说明先补什么
- 内容领域
- 使用场合
- 听众身份
- 页数和预计时长
- 证据密度
- 内容边界和不可泛化方向
- 最终交付：页面 PNG + 演讲提纲/讲稿；引用与假设说明放在演讲提纲末尾
- 页序表：页码、页面标题、页面角色、听众问题、一句话结论、核心内容关系、来源状态
- 下一步风格确认会生成哪两页样张，以及为什么选它们

推荐格式：

```text
我先按这个方向做，当前不生成图片：

- 材料状态：只有主题 / 有大纲 / 有完整文档 / 有散乱素材 / 有旧 PPT / 有数据表
- 先补策略：
- 内容领域：
- 使用场合：
- 听众身份：
- 页数/预计时长：
- 证据密度：
- 内容边界：
- 不可泛化方向：
- 最终交付：页面 PNG + 演讲提纲/讲稿

请先确认页序与内容逻辑：
| 页码 | 页面标题 | 页面角色 | 听众问题 | 一句话结论 | 核心内容关系 | 来源状态 |
| --- | --- | --- | --- | --- | --- | --- |

页序确认后，我只会先生成两张风格确认样张：
- 第 1 页：
- 第 X 页：
- 选择第 X 页的原因：
```

## 页序确认后

用户确认页序后，先只准备两张风格样张需要的内部信息，不要一次性补完全部页面细节。

最小内部文件：

- `演讲提纲.md`：使用 `assets/演讲提纲模板.md`，末尾包含引用与假设说明
- `PPT内容蓝图.md`：使用 `assets/PPT内容蓝图模板.md`，先锁定全套页序和两张样张页的完整内容
- `场景视觉简报.md`：使用 `assets/场景视觉简报模板.md`
- `页面规格.json`：使用 `assets/页面规格模板.json`，先只填写两张样张页的完整规格

等用户确认样张风格后，再补全剩余页面的 `PPT内容蓝图.md` 细节和 `页面规格.json`。

信息继承规则：

| 环节 | 继承什么 | 只新增什么 |
| --- | --- | --- |
| 材料分流与场景契约 | 用户原始请求和材料 | 材料状态、先补策略、使用场合、听众、页数、证据密度、边界 |
| 页序与内容锁定草案 | 材料分流与场景契约 | 页序、页面角色、听众问题、一句话结论、核心内容关系、来源状态 |
| PPT 内容蓝图 | 已确认页序和用户材料 | 页面任务、结论、内容块、来源状态 |
| 内容关系简报 | PPT 内容蓝图 | 每页内容如何成立、信息优先级、证据/解释/来源责任 |
| 场景视觉简报 | 内容蓝图的主题、听众、页序和内容边界 | 整套视觉世界、真实材料范式、视觉导演原则、样张选择 |
| 逐页视觉导演决策 | 内容关系简报和场景视觉简报 | 页面意图、表达策略、视觉隐喻、主视觉焦点、审美风险关注 |
| 页面规格 | 内容蓝图、内容关系简报、场景视觉简报和逐页视觉导演决策 | 逐页生成字段、图形语言提示、image 约束 |
| 图像提示词 | 页面规格里的最终字段 | 压缩成 image 可执行短 brief |

下游不重复解释上游已经决定的主题、页序、整套风格或逐页内容。

逐页视觉导演决策中的 `aesthetic_risk_focus` 最多写 3 项。不要把它写成负向规则清单；只保留本页最可能真实发生、且对设计判断有帮助的风险。

## 引用与速度

风格确认前目标是尽快进入两张样张，不是先做完整资料研究。

学术主题先建立引用框架和来源状态，不做广泛 PubMed/网页检索。引用框架至少覆盖：

- 理论或经典来源
- 方法或实验范式来源
- 关键证据来源
- 争议或局限来源

可以把具体来源标为“待核验”或“需后续核验”，但不能编造 DOI、样本量、年份、题名、精确数据或不存在的论文。

用户确认风格后，再补充引用核验，并写入 `演讲提纲.md` 的“引用与假设说明”部分。只有用户明确要求，或正式学术答辩/报告引用很多时，才额外生成 `引用与假设说明.md`。

## 风格确认

页序确认后，默认生成两张独立 PNG：

- 第 1 页：确认开场气质、标题系统和整体视觉基调
- 代表性内容页：确认内容密度、图形语言、信息层级、真实材料范式和可讲性

第二张样张选择规则：

- 学术高证据场景：机制模型页、实验范式页、数据分析管线页、证据解释页或争议边界页
- 数据复盘：指标解释页、异常归因页或决策建议页
- 内部复盘：证据墙、问题归因页或行动清单页
- 企业内训：行动路径页、场景练习页或判断矩阵页
- 课程汇报：概念解释页、案例拆解页或观点证据页

风格样张必须是独立、完整、真实尺寸的 16:9 PPT 页面 PNG。不要生成多页缩略图、风格板、mockup、截图拼排或小样合集。

两张风格样张必须属于同一套视觉系统。封面页和代表性内容页可以有不同构图，但必须共享标题层级、色彩关系、背景质感、图形/插图风格、页码页脚规则、留白节奏和装饰语言。它们不能像两套不同模板。

生成样张前，必须按 `references/page-standards.md` 的“图像调用上下文”读取并传递必要上下文：

- 读取当前成品包的 `场景视觉简报.md`
- 读取 `页面规格.json` 中第 1 页和代表性内容页的对象
- 读取 `场景视觉简报.md` 中的“套系一致性锚点”
- 把当前页的内容关系简报、视觉导演决策、必须保留的内容块和硬边界传给 image
- 不把视觉导演判断改写成大量风格命令；让 image 作为设计师决定构图、配色、字体气质和图形语言

如果第 1 页先生成，则代表性内容页必须继承第 1 页已形成的视觉系统；如果两张样张同时生成，则必须使用同一份套系一致性锚点。样张之间出现明显风格断裂时，不进入批量生成，先调整 `场景视觉简报.md` 的套系一致性锚点或重新生成样张。

生成样张后必须停止。当前回复只能包含：

- 两张样张路径
- 简短说明
- 请用户确认是否继续

## 批量生成

只有用户看到样张并明确回复“继续、满意、按这个风格生成剩余页面、可以批量生成”等，才批量生成剩余 PNG。

如果样张不满意，只调整风格提示词和页面规格；不要重做已确认页序，除非用户明确要求改内容。

批量生成前，先按已确认的风格补全剩余页面：

- 补全 `演讲提纲.md` 的讲稿准备和引用与假设说明
- 补全 `PPT内容蓝图.md` 的剩余页面内容块和来源状态
- 补全 `页面规格.json` 的剩余页面规格和逐页视觉导演决策
- 把已确认样张形成的套系一致性锚点写回 `场景视觉简报.md`，并在每页生成时继承

生成图片时再读取 `references/page-standards.md` 和 `assets/图像提示词模板.md`。图像提示词只抽取当前页必要字段，不复制整份蓝图、视觉简报或页面规格。

## 成品包

内部可以保存：

```text
成品包/
  演讲提纲.md
  PPT内容蓝图.md
  场景视觉简报.md
  页面规格.json
  页面图片/
    第01页.png
    第02页.png
```

最终回复只主展示：

```text
页面图片/
演讲提纲.md
```

只有用户明确要求单独引用文件，或正式学术答辩/报告引用很多时，才额外保存并展示 `引用与假设说明.md`。

## 检查清单

- 第一次确认是否是页序与内容锁定草案，而不是只有页标题
- 第一次确认前是否已经判断六类材料状态和场景契约
- 页序确认后是否只生成两张风格样张
- 生成样张前是否读取并传递 `场景视觉简报.md` 和当前页 `页面规格.json`
- 两张样张是否属于同一套视觉系统，而不是两套模板
- 样张确认前是否没有提前批量生成剩余页面
- 每页是否有内容关系简报和逐页视觉导演决策
- 图像提示词是否只抽必要字段，没有复制整份标准文件
- 每页 PNG 是否来自 image 模型
- `演讲提纲.md` 是否包含引用与假设说明
- 来源、引用、模拟假设是否区分清楚
- 没有编造论文、数字、样本量或真实数据
- 最终回复是否只主展示 `页面图片/` 和 `演讲提纲.md`
