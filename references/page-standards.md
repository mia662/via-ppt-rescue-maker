## 核心原则

把 image 模型当作资深演示设计师、信息设计师和视觉编辑来委托，而不是当作模板填充工具。

skill 只负责给出：

- 这是什么内容领域
- 用在什么场合
- 谁会看
- 这一页要完成什么沟通任务
- 哪些内容必须保留
- 哪些结果明显不合格

具体视觉语言、配色、构图、图形方式和质感由 image 模型根据场景决定。不要替 image 模型规定固定槽位、左右结构、卡片数量或统一模板。

页面要像真实用户明天就能拿去汇报的材料，不像临时模板、营销海报、公众号封面、素材拼贴或 AI 试手图。

## 正向审美北极星

每套 PPT 生成图片前，先确定一个正向审美北极星：这套页面在当前场景里应该接近哪类优秀真实材料。

必须回答：

- 这套 PPT 最像哪类真实材料
- 为什么它适合当前内容领域、使用场合和听众身份
- 页面靠什么建立美感：机制关系、证据组织、案例空间、数据对比、行动路径或课堂解释
- 哪些页面最能代表整套视觉质量，应该作为风格校准页

防错规则只能用来排除明显失败，不能替代正向审美北极星。重生成时优先改“这套 PPT 应该像什么优秀材料”，不要只堆新的禁令。

## 信息继承

上游决定过的内容，下游只引用，不重新解释。

- 材料分流与场景契约决定材料状态、先补策略、使用场合、听众、页数、证据密度和内容边界
- 页序与内容锁定草案决定页面顺序、页面角色、听众问题、一句话结论、核心内容关系和来源状态
- 内容蓝图决定主题、页面任务、结论、内容块和来源状态
- 内容关系简报决定每页内容如何成立、信息优先级、证据/解释/来源责任
- 场景视觉简报决定整套视觉世界和真实材料范式
- 逐页视觉导演决策决定页面意图、表达策略、视觉隐喻、主视觉焦点和审美风险关注
- 页面规格补齐逐页生成字段、视觉导演决策和图形语言提示
- 图像提示词只抽取必要字段，压缩成 image 可执行 brief

如果多个环节都在重复写“专业、高级、不要模板、信息密但有秩序”，说明分层失败。应回到各层独有职责，删掉重复解释。

## 视觉导演决策

视觉导演决策把“讲什么”和“像什么材料”翻译成逐页表达判断。它不是新的交付文件，而是写入 `页面规格.json` 的每页字段。

每页必须回答：

- 页面意图：要让观众感受到或理解什么
- 表达策略：用什么视觉策略表达内容关系
- 视觉隐喻：可选；没有必要时写“不使用隐喻，使用信息结构表达”
- 主视觉焦点：焦点来自机制、证据、对比、行动路径、案例空间或其他内容关系
- 审美风险关注：最多 3 个本页最需要警惕的风险

视觉导演决策只做正向表达判断，不规定固定布局、左右结构、卡片数量、具体坐标或版式槽位。

## 真实材料范式

场景适配靠真实材料范式，不靠“专业、高级、好看”这类抽象词。

先拆三轴：

- 内容领域：这页真正讲什么知识、业务、行为或证据
- 使用场合：组会、答辩、课程、内训、复盘、策略会等
- 听众身份：学习者、评审者、执行者、决策者、客户、家长等

常见范式：

| 场景 | 更应该接近 |
| --- | --- |
| 研究组会 | 机制综述图、实验范式页、证据解释页 |
| 开题/答辩 | 研究问题页、方法架构页、可行性论证页 |
| 课程汇报 | 课堂解释页、案例拆解页、概念图解页 |
| 企业内训 | 工作坊讲义页、行动建议页、场景练习页 |
| 决策者汇报 | 决策摘要页、指标解释页、风险动作页 |
| 内部复盘 | 证据墙、问题归因页、行动清单页 |
| 数据复盘 | 指标看板、异常解释页、决策建议页 |
| 内容策略 | 用户场景页、案例板、流程计划页 |
| 低龄教学 | 活动页、具象解释页、互动任务页 |

这些范式不是固定模板，只是给 image 的正向锚点。

## 学术高证据场景

当内容领域是科研、医学、心理学、神经科学、工程研究、论文汇报、开题答辩、组会或 journal club，且听众是导师、评审、研究生、同行或专业学习者时，可以启用学术高证据正向样本。

这类页面应该接近：

- 机制综述图：用核心机制图统领变量、路径、信号和结论
- 实验范式页：任务、条件、变量、时间线和输出指标同屏组织
- 数据分析管线：输入、预处理、指标提取、统计模型和结果输出形成流程
- 证据解释页：图表、指标、解释范围、局限和下一步验证同时出现
- 图形摘要式页面：主图居中，证据、方法、结论和来源围绕主图展开

这套样式只适用于学术高证据场景。非学术场景必须重新选择自己的真实材料范式。

## 内容关系简报

内容关系简报是内容蓝图和 image 生成之间的中间层。它只说明内容如何成立，不规定版式。

每个内容页必须先定义：

- 听众问题：这页回答什么
- 一句话结论：这页要让听众相信什么
- 核心内容关系：机制、流程、对比、证据、案例、指标或行动路径
- 内容块功能：定义、变量、证据、解释、限制、行动或来源
- 信息优先级：哪些内容最重要，哪些是辅助解释
- 来源责任：哪些来自用户材料、引用、计算、模拟假设或待核验

内容关系要稳定“可讲性”，但排版、比例、分区、留白、图形表达由 image 模型决定。

## 信息密度

信息密度服从页面角色。

- 高证据场景：研究组会、开题答辩、数据复盘、内部复盘、策略判断、专业培训等页面，应像可以讲 2 到 5 分钟的真实汇报页，保留问题、机制、变量、证据、方法、解释、限制或行动中的多层结构。
- 中等解释场景：课程汇报、概念讲解、方法介绍等页面，可以更清爽，但仍要有概念关系、案例、判断依据或讲述线索。
- 低认知负担场景：低龄教学、公众科普、活动说明等页面，可以主动降低信息密度，但仍要服务理解。

重点页要有可讲的复杂度：一个核心内容关系，三到六个有效内容块，至少一条推理线索、证据线索或行动线索。复杂度不等于拥挤；留白要服务层级。

## 设计完成度

生成后检查：

- 是否有明确视觉焦点，而不是所有模块平均用力
- 信息层级是否自然，标题、结论、主体、注释不会互相抢
- 留白和边距是否让页面能呼吸
- 色彩是否服务内容语义和场景情绪
- 中文字形比例是否自然，没有被压缩、拉伸或挤瘦
- 图形是否在解释内容，而不是只做装饰
- 整页是否像同一位专业设计师完成的真实 PPT 页面

设计感来自信息层级、比例、留白、对齐、字体气质、色彩语义和图形解释力，不来自装饰数量。

## 套系一致性

风格确认不是只看单页是否好看，而是确认整套 PPT 的视觉系统能否成立。封面页、代表性内容页和后续页面不需要同版式，但必须像同一套材料。

必须跨页继承：

- 字体气质和标题层级
- 主色、辅助色、强调色的关系
- 背景质感或页面底色
- 图形、插图、图标、线条的风格
- 圆角、标签、注释、分隔线、装饰元素的处理方式
- 页码、页眉、页脚和来源说明的规则
- 留白节奏、边距气质和整体成熟度

可以按页面任务变化：

- 构图和主视觉位置
- 图表、流程、矩阵、案例、机制或行动路径的具体表达
- 信息密度和内容块数量
- 是否使用图片、插图或图形摘要

如果封面像生活方式科普页，内容页却变成企业蓝白流程模板；或者封面是柔和插画系统，内容页变成硬朗商务图标系统，说明套系一致性失败。

## 图像调用上下文

生成图片时，image 模型不能默认自己读取本地文件路径；执行者必须在调用 image 前读取当前成品包里的上下文文件，并把必要内容作为调用上下文提供给 image。

每页图片调用前必须读取：

- `场景视觉简报.md`：读取基本判断、真实材料范式、视觉导演原则、视觉系统、套系一致性锚点和样张选择。
- `页面规格.json`：读取当前页对象，尤其是 `content_relationship_brief`、`visual_director_decision`、`graphic_language_hint`、`content_blocks`、`image_prompt_constraints`。

这些内容不是用来替 image 模型做排版决定，而是让 image 理解任务、受众、内容关系、视觉导演判断和锁定内容。不要把文件整段复制成冗长提示词，也不要把视觉导演判断改写成大量“应该/不要”的风格命令。

调用 image 时只需要传递：

- 当前场景、听众和页面任务
- 当前页标题、结论、必须保留的内容块和来源说明
- 当前页内容关系简报
- 当前页视觉导演决策
- 套系一致性锚点；批量生成时还要继承用户确认过的样张风格
- 必要的硬边界：单页 16:9、中文可读、锁定内容不改写、不编造数据或引用

image 模型作为设计师自行决定构图、图形语言、配色、字体气质、留白和视觉表达。执行者不要用固定槽位、卡片数量、左右结构或过度具体的美术指令替代设计判断。

## 首图确认

风格确认必须生成真实单页 PNG，不是风格板、缩略图、mockup 或多页拼排。

默认生成两张独立样张：

- 第 1 页：确认开场气质、标题系统和整体视觉基调
- 代表性校准页：确认内容密度、图形语言、信息层级、真实材料范式和可讲性

如果第 1 页是封面，第二张必须是重点内容页。封面不能作为唯一风格判断依据。

展示样张前自检：

- 主题、标题、结论和核心内容没有被泛化或改写
- 内容密度匹配场景
- 主视觉来自内容关系
- 标题、正文、页码、页脚完整可读
- 风格能支撑整套页面继续生成

生成样张后必须停止，等待用户确认是否继续批量生成。

## 失败边界

以下结果需要重生成：

- 文字、标题、页码或页脚被裁切
- 中文文字被横向压缩、拉伸或挤瘦
- 锁定主题被改写、泛化、扩大或替换
- 首图是多页缩略图、风格板、mockup、截图拼排或小样合集
- 页面变成封面、海报、公众号配图、营销主图或素材拼贴
- 成人专业场景被做成儿童科普、亲子绘本、可爱贴纸或情绪治愈卡片
- 高证据或专业场景被压成简明知识卡、单张大插图加列表，缺少可讲结构
- 页面全靠等宽卡片、粗框、编号圆点、通用图标和大箭头堆起来
- 主视觉是无关氛围照片，而不是证据、案例或内容关系
- 图表没有变量名、单位、对照组或解释
- 质感油亮、塑料、过度磨皮、假 3D、糖果渐变或随机装饰过多

儿童绘本感、卡通人物、圆润图标和柔和高饱和色不是绝对禁止；只有当受众、场合和沟通任务明确需要时才使用。

## 提示词使用

实际生成图片时使用 `assets/图像提示词模板.md` 的短模板。

本文件只做质量判断和自检，不要整段复制进 image 提示词。生成提示词时只抽取当前页必要字段：正向审美北极星、页面任务、内容关系简报、视觉导演决策、锁定内容、推荐图形语言、设计委托和少量硬失败边界。
