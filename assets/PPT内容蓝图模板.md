## 内容关系简报规则

每个内容页都要先写内容关系简报，再写具体内容。它用于告诉 image 模型“内容如何成立”，不是规定版式槽位。

- 页面回答的听众问题：
- 一句话结论：
- 核心内容关系：机制 / 流程 / 对比 / 证据 / 案例 / 指标 / 行动路径
- 内容块功能：定义 / 变量 / 证据 / 解释 / 限制 / 行动 / 来源
- 信息优先级：
- 证据、解释、来源或假设责任：

常用内容关系类型。只描述讲述逻辑，不规定左右、上下、卡片数量或固定构图：

- 机制页：问题、核心机制、变量/信号、假设或预测、来源
- 实验/方法页：任务、条件/分组、变量、时间线、输出指标
- 数据/结果页：结论、主证据、对照/统计、解释、边界
- 复盘/证据页：事实、证据、归因、影响、动作
- 行动/训练页：场景、判断标准、行动路径、风险提醒、练习
- 概念/课程页：概念、案例、对比、误区、小结

- 主题是否具体到一个可讲清的问题：
