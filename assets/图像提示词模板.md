# 图像提示词模板

生成图片时使用短提示词。调用前必须先读取当前成品包的 `场景视觉简报.md` 和 `页面规格.json` 当前页对象，但不要把这些文件整段复制进提示词。

每页提示词只填必要字段，建议控制在 600 到 1000 个中文字符以内。目标是让 image 模型拿到任务、内容、约束和视觉导演判断，然后作为设计师自行完成视觉表达。

```text
生成一张完整 16:9 演示文稿页面图片。

你是资深演示设计师、信息设计师和视觉编辑。请基于给定场景、内容关系和视觉导演判断，自行决定构图、信息层级、图形语言、配色、字体气质和留白，把锁定内容转译成一页可直接汇报的 PPT 页面。

场景：
- 内容领域：[例如 神经科学 / 企业内训 / 数据复盘 / 课程汇报]
- 使用场合：[例如 研究生组会 / 开题答辩 / 内部复盘 / 工作坊]
- 听众身份：[例如 导师和研究生 / 评审 / 执行者 / 客户]
- 视觉导演上下文：[来自场景视觉简报，简述真实材料范式、视觉系统和重要风险；不要写成长篇风格命令]
- 套系一致性锚点：[来自场景视觉简报；说明跨页必须继承的字体气质、色彩关系、背景质感、图形/插图风格、页码页脚规则和留白节奏]

页面任务：
- 页码：[第 XX 页]
- 页面标题：[锁定标题]
- 一句话结论：[锁定结论]
- 页面角色：[解释机制 / 展示范式 / 分析数据 / 拆解案例 / 给出行动]

内容关系简报：
- 核心内容关系：[机制 / 流程 / 对比 / 证据 / 案例 / 指标 / 行动路径]
- 信息优先级：[最重要信息 / 辅助解释 / 补充边界或来源]
- 证据/解释/来源责任：[一句话说明它们在本页承担什么作用]

视觉导演决策：
- 页面意图：[这一页要让观众感受到或理解什么]
- 表达策略：[用什么视觉策略表达内容关系]
- 视觉隐喻：[可选；没有必要时写“不使用隐喻，使用信息结构表达”]
- 主视觉焦点：[焦点来自机制、证据、对比、行动路径、案例空间或其他内容关系]
- 审美风险关注：[最多 3 个；作为设计注意事项，不写成禁令清单]

锁定内容：
[只写本页必须出现的 3 到 6 个内容块，并标注每块功能：定义、变量、证据、解释、限制、行动或来源。不要粘贴整份大纲。]

推荐图形语言：
[机制图 / 流程图 / 时间线 / 对照矩阵 / 数据图 / 证据墙 / 行动板 / 由设计师基于内容选择]

设计委托：
请自主完成页面设计，不使用固定槽位、固定卡片数量或预设左右结构。页面必须保留锁定内容，并让内容关系成为主视觉来源。若已有确认样张，请让本页继承确认样张的视觉系统；允许根据页面任务改变构图，但不要切换成另一套模板、另一种插图/图标系统或另一套色彩逻辑。

硬失败边界：
- 只生成这一页，不生成多页缩略图、风格板、mockup 或拼贴预览。
- 保持锁定标题、结论和核心内容，不改写主题，不编造精确数据或引用。
- 中文字形自然，标题、正文、页码、页脚完整可读。
```
